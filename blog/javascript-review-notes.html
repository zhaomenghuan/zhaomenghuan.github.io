JavaScript 进阶之基础问题 ​

前言 ​

JavaScript 是一门灵活的语法，对于很多人包括我而言在平时工作中可能会用到一些新特性，又或者用着老语法也能实现某个功能，但是一直没有系统性的梳理更新自己的知识体系，时间长了总觉得有些内容能够按照某种方式去实现，但是并没有去思考为什么是这样做，或者还有没有更好的办法去实现。本系列文章主要想通过对于前端开发过程中最经典的内容进行梳理总结。

变量与作用域 ​

JavaScript 中声明变量的方法：

  • ES5：var、function
  • ES6：新增 let、const、import、class

变量定义与声明 ​

1.ES6 中 let 与 var 的区别？

  • 用 let 重定义变量会抛出一个语法错误（SyntaxError）。

let 不允许在相同作用域内，重复声明同一个变量。

js
// 报错
function func() {
  let a = 10;
  var a = 1;
}

// 报错
function func() {
  let a = 10;
  let a = 1;
}
  • let 声明的变量拥有块级作用域。

JS 在 ES6 之前是遵循函数作用域，不支持块级作用域，ES6 中 let 声明的变量的作用域只是外层块，而不是整个外层函数，并且 let 声明的全局变量不是全局对象的属性。这就意味着，你不可以通过 window.变量名 的方式访问这些变量。它们只存在于一个不可见的块的作用域中，这个块理论上是 Web 页面中运行的所有 JS 代码的外层块。

js
// ES5
var i = 0;
if (i < 2) {
  var i = 2;
}
console.log(i); // 2
console.log(window.i); // 2

// ES6
let i = 0;
if (i < 2) {
  let i = 2;
}
console.log(i); // 0
console.log(window.i); // undefined
  • let 声明的变量直到控制流到达该变量被定义的代码行时才会被装载，所以在到达之前使用该变量会触发错误。

JS 存在变量提升和函数提升的问题。变量提升和函数提升：就是将变量声明或者函数全部代码提升到当前作用域（全局作用域或函数作用域）最开始的部分。JavaScript 仅提升声明，而不提升初始化。

js
// 变量类型：
foo = 2;
var foo;

// 被隐式地解释为:
var foo;
foo = 2;

// 函数类型：
hoisted(); // "foo"
function hoisted() {
  console.log("foo");
}

对于 let 声明的变量不允许变量提升，如果一个没有经过声明的变量被引用，则会直接抛出 ReferenceError 异常。不可访问的这段时间变量一直处于作用域中，但是尚未装载，它们位于临时死区（Temporal Dead Zone，简称 TDZ）中。

js
console.log(varValue); // undefined
varValue = 2;
var varValue;
console.log(varValue); // 2

console.log(letValue); // Uncaught ReferenceError: letValue is not defined
letValue = 2;
let letValue;
console.log(letValue);
  • 形如 for (let x...) 的循环在每次迭代时都为 x 创建新的绑定。

for 循环内 var 声明的变量存在过度共享，会导致异常，如下：

js
var messages = ["one", "two", "three"];
for (var i = 0; i < messages.length; i++) {
  console.log("fun outside: " + i);
  setTimeout(function() {
    console.log("fun inside: " + i);
    console.log(messages[i]);
  }, 0);
}

// fun outside: 0
// fun outside: 1
// fun outside: 2
// fun inside: 3
// undefined
// fun inside: 3
// undefined
// fun inside: 3
// undefined

这个问题的答案是，循环本身及三次 timeout 回调均共享唯一的变量 i，当循环结束执行时，i 的值为 3。解决这个问题有很多种方法，例如：

js
var messages = ["one", "two", "three"];
messages.forEach(function(message, i) {
  setTimeout(function() {
    console.log(i);
    console.log(message);
  }, 0);
});

// 0
// one
// 1
// two
// 2
// three

但是上述问题使用 let 即可解决 for 循环内变量过度共享的 bug。

js
var messages = ["one", "two", "three"];
for (let i = 0; i < messages.length; i++) {
  setTimeout(function() {
    console.log(i);
    console.log(messages[i]);
  }, 0);
}

变量的解构赋值 ​

ES6 允许按照一定模式，从数组和对象中提取值，对变量进行赋值，这被称为解构（Destructuring）。ES5 为变量赋值，只能直接指定值，ES6 新增变量的解构赋值。解构赋值的规则是，只要等号右边的值不是对象或数组，就先将其转为对象。由于 undefined 和 null 无法转为对象，所以对它们进行解构赋值，都会报错。

js
let { prop: x } = undefined; // TypeError
let { prop: y } = null; // TypeError
  • 1.数组的解构赋值
js
let [a, b, c] = [1, 2, 3];
// a = 1; b = 1; c = 1;
  • 2.对象的解构赋值
js
let { foo } = { foo: "aaa", bar: "bbb" };
// foo = "aaa"

let { name: username } = { name: "JavaScript" };
// username = "JavaScript"
  • 3.字符串的解构赋值

字符串被转换成了一个类似数组的对象。

js
let [a, b, c, d, e] = "hello";
// a = "h"; b = "e"; c = "l"; d = "l"; e = "o";

类似数组的对象都有一个 length 属性，因此还可以对这个属性解构赋值。

js
let { length: len } = "hello";
// len = 5;
  • 4.数值和布尔值的解构赋值

解构赋值时，如果等号右边是数值和布尔值，则会先转为对象。

js
let { toString: s } = 123;
s === Number.prototype.toString; // true

let { toString: s } = true;
s === Boolean.prototype.toString; // true
  • 5.函数参数的解构赋值
js
function add([x, y]) {
  return x + y;
}
add([1, 2]); // 3

[[1, 2], [3, 4]].map(([a, b]) => a + b);
// [ 3, 7 ]

变量的解构赋值用途：

  • 交换变量的值
js
let x = 1;
let y = 2;

[x, y] = [y, x];
  • 输入模块的指定方法
js
import { map } from "./utils";
  • 提取 JSON 数据
js
let userData = {
  code: 0,
  message: "请求成功",
  data: {
    id: "49a9dd505c9d11e8b5e86b9776bb3c4f",
    username: "zhaomenghuan",
    email: "1028317108@qq.com"
  }
};

let {
  data: { id: userKey, username, email }
} = userData;
// userKey = "49a9dd505c9d11e8b5e86b9776bb3c4f"
// username = "zhaomenghuan"
// email = "1028317108@qq.com"
  • 从函数返回多个值

函数只能返回一个值，如果要返回多个值，只能将它们放在数组或对象里返回。

js
function getArrayResult() {
  return [1, 2, 3];
}
let [a, b, c] = getArrayResult();
  • 函数参数的定义

解构赋值可以方便地将一组参数与变量名对应起来。

js
// 参数是一组无次序的值
function f({x, y, z}) { ... }
f({z: 3, y: 2, x: 1});
  • 函数参数的默认值
js
function getPageList(page = 1, pageSize = 10) {...}

指定参数的默认值，就避免了在函数体内部再写 page = page || 1; 这样的语句。

  • 遍历 Map 结构
js
const map = new Map();
map.set("name", "zhaomenghuan");
map.set("site", "https://zhaomenghuan.github.io/");

for (let [key, value] of map) {
  console.log(key + " is " + value);
}

数据类型和数据结构 ​

JavaScript 是一种弱类型或者说动态语言。这意味着你不用提前声明变量的类型，在程序运行过程中，类型会被自动确定。

JavaScript 数据类型分类 ​

1.JS 有哪几种数据类型？

最新的 ECMAScript 标准定义了 7 种数据类型:

js
Null、Undefined、Number、Boolean、String、Symbol、Object

JS 中只有六种原始类型和一个 Object（简记为"6+1"），Symbol 是 ECMAScript 6 新定义，对于 Function、Array、Date、JSON、Map、Set 这种都属于 Object。

2.为什么 typeof null 是 object？

我们可以使用 typeof 操作符判断对象类型，typeof 可能的返回值:

类型结果
Undefined"undefined"
Null"object"
Boolean"boolean"
Number"number"
String"string"
Symbol"symbol"
函数对象"function"
宿主对象（由 JS 环境提供）Implementation-dependent
任何其他对象"object"

在 JavaScript 最初的实现中，JavaScript 中的值是由一个表示类型的标签和实际数据值表示的。对象的类型标签是 0。由于 null 代表的是空指针（大多数平台下值为 0x00），因此，null 的类型标签也成为了 0，typeof null 就错误的返回了"object"。

3.怎么完整判断一个对象类型？

Array 对象我们也可以使用 Array.isArray 判断。

对于原始类型我们可以通过 typeof 基本判断出基本类型（除了 null 返回的是 object），但是很显然对于 Object 我们通过 typeof 得到的基本都是 "object"，所以 typeof 并不能得到预期的数据类型，所以我们通常会使用下面的办法判断：

js
const objectType = object => {
  return Object.prototype.toString
    .call(object)
    .slice(8, -1)
    .toLowerCase();
};

测试如下：

js
typeof JSON;
// "object"
objectType(JSON);
// "json"

typeof /a/;
// "object"
objectType(/a/);
// "regexp"

相等性与存在性判断 ​

JavaScript 提供三种不同的值比较操作：

  • 严格相等 ("triple equals" 或 "identity")，使用 ===
  • 宽松相等 ("double equals") ，使用 ==
  • 以及 Object.is （ECMAScript 2015/ ES6 新特性）

ES5 比较两个值是否相等，只有两个运算符：相等运算符（==）和严格相等运算符（===）。它们都有缺点，前者会自动转换数据类型，后者的 NaN 不等于自身，以及+0 等于-0。JavaScript 缺乏一种运算，在所有环境中，只要两个值是一样的，它们就应该相等。ES6 提出 "Same-value equality"（同值相等）算法，它用来比较两个值是否严格相等，与严格比较运算符（===）的行为基本一致。

js
NaN === NaN;
// false
Object.is(NaN, NaN);
// true
js
+0 === -0;
// true
Object.is(+0, -0);
// false
js
Object.is("1", "1");
// true
Object.is(undefined, null);
// false

 认识了 JS 中值相等的比较，那么我们就可以聊聊 JS 中值传递和引用传递的问题。首先看一个典型例子is-javascript-a-pass-by-reference-or-pass-by-value-language：

js
function changeStuff(a, b, c) {
  a = a * 10;
  b.item = "changed";
  c = { item: "changed" };
}

var num = 10;
var obj1 = { item: "unchanged" };
var obj2 = { item: "unchanged" };

changeStuff(num, obj1, obj2);

console.log(num);
console.log(obj1.item);
console.log(obj2.item);

// 10
// changed
// unchanged
